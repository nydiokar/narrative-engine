# Workflows Overview

The narrative pipeline splits into **8 workflows**, each producing specific contract types.

## Workflow Progression

```
00-brief-and-taxonomy  →  story, theme, character, world
01-seed-to-premise     →  character, object_of_value
02-premise-to-structure →  world
03-structure-to-episodes →  episode, chapter
04-episodes-to-scenes  →  scene
05-scenes-to-draft     →  discourse
06-editorial-passes    →
07-critique-and-revision →  critique
```

## Workflow Details

| Workflow | Steps | Outputs |
|----------|-------|---------|
| [00 — Brief & Taxonomy](00-brief-and-taxonomy.md) | theme → genre → world → character → approve | Brief ready |
| [01 — Seed to Premise](01-seed-to-premise.md) | review → analyze → backbone → protagonist → approve | Premise with actantial model |
| [02 — Premise to Structure](02-premise-to-structure.md) | fabula → constraints → thematic fit → approve | Structure validated |
| [03 — Structure to Episodes](03-structure-to-episodes.md) | segment → chapters → arcs → settings → approve | 4 episodes, chapters |
| [04 — Episodes to Scenes](04-episodes-to-scenes.md) | enact → dialogue → render → diagnostic → continuity | Scenes with Greimas diagnostics |
| [05 — Scenes to Draft](05-scenes-to-draft.md) | finalize → check → assemble | Draft assembled |
| [06 — Editorial Passes](06-editorial-passes.md) | dev edit → structural fix → line edit → copy edit → proof | Passed editorial |
| [07 — Critique & Revision](07-critique-and-revision.md) | hard gate → soft gate → greimas → revision → approve | Final approval |

Medium-specific rendering (WF 04–06) differs per medium type. See each workflow page for details.
