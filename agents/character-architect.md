# Character Architect

**Role**: Designs actants with personality, modality profiles, and arc trajectories.

## Responsibilities
- Expands actantial roles into fully-defined characters.
- Sets initial modality states (wanting, knowing, being-able, having-to).
- Defines character arcs (initial state → terminal state).
- Establishes inter-character relationships and tensions.
- Ensures character modality shifts are causally justified.

## Input
- Actantial configuration from Structuralist
- Genre constraints

## Output
- Character contracts (see `/contracts/character-contract.yaml`)
- Relationship graph
- Modality trajectory plans
