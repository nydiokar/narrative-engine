# Workflow 07: Critique and Revision

**Goal**: Evaluate the draft through the two-gate system and iterate until passing.

## Steps

1. **Critic Runs Hard Gate**
   - Checks causal soundness, character intentionality, world rule consistency, stakes presence, contradiction-free, conflict active, continuity.
   - **If any fail**: immediate rejection with specific violation report.

2. **Critic Runs Greimas Fabula Coherence Engine**
   - 8 diagnostic checks (state transformation, value-object clarity, object-desire, opponent value-logic, qualification/performance/sanction completeness, actantial position change, supported heroicity).
   - Flags cliché patterns.

3. **Critic Runs Soft Gate**
   - Scores: genre fit, thematic clarity, conflict density, relationship tension, scene-level purpose, suspense/curiosity/surprise, emotional transport, novelty, prose distinctiveness.
   - Computes weighted composite score.

4. **If Pass (hard gate clear + soft gate above threshold)**: Workflow complete. Proceed to release package.

5. **If Fail**:
   - Revision Agent parses Critique Contract
   - Prioritizes hard gate failures first, then soft gate weaknesses
   - Applies targeted revisions
   - Returns revised artifact for re-assessment
   - Loop continues until all hard gates pass and soft gate clears threshold

6. **Showrunner Approves Final Version**
   - Signs off on final draft
   - Updates story status

## Output
- Final narrative draft (passing all gates)
- Critique Contract (archive)
- Revision log
- Ready for Release Package (blurb, metadata, BISAC tags, series bible update)
